# 电力负荷预测系统 (Power Load Prediction)

基于 XGBoost 集成学习算法的电力负荷预测项目，用于预测未来时刻的电力负荷值。

## 项目简介

本项目利用机器学习技术对历史电力负荷数据进行分析和建模，通过 XGBoost 回归模型实现电力负荷的精准预测。项目包含完整的数据预处理、特征工程、模型训练、评估、预测及可视化分析流程。

## 使用说明

### 1. 数据准备

将训练数据和测试数据分别放置在 `data/` 目录下：
- `train.csv`: 用于模型训练的历史电力负荷数据
- `test.csv`: 用于模型测试和预测的数据

数据格式要求：CSV 文件应包含 `time`（时间戳）和 `power_load`（电力负荷值）两列。

### 2. 模型训练

运行训练脚本以构建预测模型：

```bash
cd src
python train.py
```

训练过程包括：
- 数据预处理（时间格式化、排序、去重）
- 特征工程（时间 One-Hot 编码、滞后特征构造）
- 网格搜索与交叉验证寻找最优超参数
- 模型评估（MSE、RMSE、MAE、MAPE）
- 模型保存至 `model/model.pkl`

### 3. 模型预测

使用训练好的模型进行预测：

```bash
cd src
python predict.py
```

预测流程：
- 加载预训练模型
- 模拟实际场景逐步预测（仅使用预测时刻之前的历史数据）
- 可视化实际值与预测值对比图
- 计算并记录预测性能指标

### 4. 可视化分析

生成论文级高质量图表（300 DPI）：

```bash
cd src
python visualization.py
```

包含以下 5 类分析图表（详见下方"可视化分析"章节）。

## 核心功能

### 特征工程

项目实现了丰富的特征提取策略：

1. **时间特征**:
   - 小时特征（24 维 One-Hot 编码）
   - 月份特征（12 维 One-Hot 编码）

2. **滞后特征**:
   - 前 1 小时负荷 (Lag_1h)
   - 前 2 小时负荷 (Lag_2h)
   - 前 3 小时负荷 (Lag_3h)
   - 昨日同时刻负荷 (Lag_24h)

### 模型训练与评估

- **超参数优化**: 网格搜索 (GridSearchCV) + 5 折交叉验证，搜索空间覆盖 `n_estimators`、`learning_rate`、`max_depth`
- **评估指标**:
  - 均方误差 (MSE)
  - 均方根误差 (RMSE)
  - 平均绝对误差 (MAE)
  - 平均绝对百分比误差 (MAPE)

### 日志系统

项目内置完善的日志记录机制：
- 训练日志：记录模型训练过程中的关键信息（最优参数、评估结果、模型保存路径）
- 预测日志：记录预测执行过程和性能指标
- 日志文件按日期命名，便于追踪和管理

## 可视化分析

项目提供三类可视化：

### 1. 数据探索分析 (`train.py` → `analysis_data()`)

- 负荷整体分布直方图

### 3. 模型分析可视化 (`visualization.py`)

|------|------|------|
| 1 | 滞后特征自相关分析图 (ACF) | 负荷序列在不同滞后步长上的自相关系数，含 95% 置信界，标注 24h/48h/168h 关键周期 |

## 注意事项

1. **时间序列特性**: 本项目采用时间顺序划分数据集（80% 训练集，20% 验证集），避免随机划分导致的数据泄露问题。

2. **特征一致性**: 预测时的特征提取必须与训练时保持一致，确保模型输入格式正确。

3. **缺失值处理**: 对于历史数据中不存在的时刻，使用默认值 500 进行填充。

4. **中文显示**: 图表使用 SimHei 字体，字体配置需在 `sns.set_style()` 之后设置，否则会被 seaborn 覆盖。若中文无法正常显示，请检查系统是否已安装 SimHei 字体。

5. **预测模拟**: 预测脚本模拟真实应用场景——每次预测仅使用该时刻之前的历史数据，避免未来信息泄露。
